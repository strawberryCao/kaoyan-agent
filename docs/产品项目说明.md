# Kaoyan Agent 产品项目说明

## 一句话定位

Kaoyan Agent 是一个面向考研备考的学习 Agent。它不是普通聊天机器人，而是围绕“发现问题、记录证据、形成记忆、推动下一步行动”的备考辅助系统。

## 目标用户

目标用户是正在准备考研的学生，尤其适合需要长期跟踪学习状态、错题原因、任务执行和复盘结果的用户。

系统重点解决三类问题：

- 学习过程信息分散，聊天、任务、错题、成绩、复盘没有连起来。
- 用户知道自己“学得不好”，但很难结构化地定位具体问题。
- 每天产生很多学习记录，但缺少长期记忆和后续干预。

## 产品核心价值

Kaoyan Agent 的核心不是“陪聊”，而是把学习过程变成可追踪的数据闭环：

```text
用户输入 / 学习记录
-> 原始事件和对话沉淀
-> 晚间结构化复盘
-> 问题发现与长期记忆
-> 任务、错题、督学、激励等后续行动
```

也就是说，Chat 是入口，问题板和长期记忆才是系统持续变聪明的基础。

## 当前 MVP 功能

当前产品是单一考研备考工作区，不做多空间管理。侧边栏结构如下：

```text
Kaoyan Agent
├─ + 新建对话
├─ 今日任务 / 学习规划
├─ 督学模式
├─ 错题复习
├─ 成绩趋势
├─ 最近会话
├─ 晚间回顾
├─ 问题板
├─ 运势签
└─ 设置
```

### 新建对话

用户可以随时开始一段新的学习对话。对话内容会被保存，后续可以被晚间回顾读取，用于发现问题和更新记忆。

### 今日任务 / 学习规划

展示和生成学习任务，帮助用户把复习目标拆成可执行动作。

### 督学模式

提供番茄钟式专注记录和任务执行跟踪。它是独立入口，不依赖任何“工作台”或多空间切换。

### 错题复习

记录错题、错因、知识点和复刷优先级，让错题从“做错了”变成可追踪的复习对象。

### 成绩趋势

记录分数和科目表现，帮助用户观察长期变化。

### 晚间回顾

晚间回顾会读取当天对话、事件、任务、错题、成绩等信息，调用 LLM 做结构化总结。它会产出关键事件、发现的问题、记忆更新和下一步行动。

### 问题板

问题板展示系统发现的学习问题，例如拖延、概念薄弱、错题复刷不足、计划执行断裂等。问题板是后续干预的基础。

### 运势签

运势签是轻量激励功能，给用户一个低压力、可立即执行的小动作。

### 设置

设置页展示当前记忆、模型配置和数据库路径。它不是复杂后台，只保留对 MVP 有用的信息。

## 产品边界

当前版本明确不做这些事情：

- 不做多项目、多空间、多工作区管理。
- 不把“项目”作为用户可见产品概念。
- 不把 Chat 当作唯一核心能力。
- 不直接相信 LLM 原始输出写入关键数据表。

历史数据库中可能仍有 `projects` 表和 `project_id` 字段，但它们只是兼容旧数据，不是当前产品设计。

## 典型演示流程

可以这样向老师或组员演示：

1. 打开系统，展示侧边栏：这是单一考研备考 Agent。
2. 点击 `+ 新建对话`，输入今天的学习情况。
3. 进入 `今日任务 / 学习规划`，展示任务如何被记录和推进。
4. 进入 `督学模式`，展示专注记录。
5. 进入 `错题复习`，展示错因和复刷优先级。
6. 进入 `晚间回顾`，运行结构化复盘。
7. 进入 `问题板`，展示系统如何把聊天和学习记录变成可跟踪问题。
8. 进入 `设置`，展示长期记忆和当前运行配置。

## 给组员的产品口径

可以这样讲：

“我们做的是一个考研问题发现 Agent。用户白天通过聊天、任务、错题、督学记录学习过程；晚上系统把这些信息结构化，发现备考问题，更新长期记忆，并生成下一步行动。我们的重点不是做一个漂亮聊天框，而是做一套从学习证据到问题发现再到干预行动的闭环。”
