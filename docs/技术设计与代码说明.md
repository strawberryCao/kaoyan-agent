# Kaoyan Agent 技术设计与代码说明

## 总体架构

项目采用 Python `src layout`：

```text
app.py
-> src/kaoyan_agent/ui/
-> src/kaoyan_agent/workflows/
-> src/kaoyan_agent/agents/ + services/ + repositories/
-> src/kaoyan_agent/db/
```

根目录只保留启动入口、文档、测试、配置样例和本地数据目录。正式业务包只有一个：

```text
src/kaoyan_agent/
```

## 入口层

`app.py` 是唯一 Streamlit 启动入口，职责很窄：

- 把 `src/` 加入 Python import path。
- 调用 `init_db()` 初始化或兼容迁移数据库。
- 初始化 Streamlit 页面状态。
- 渲染侧边栏。
- 按 `current_main_view` 分发到具体页面。

当前 Streamlit 主状态只需要：

```text
current_main_view
current_chat_session_id
```

不要在 UI 中恢复 `current_project_id` 或类似状态。

## UI 层

位置：

```text
src/kaoyan_agent/ui/
src/kaoyan_agent/ui/components/
```

页面文件负责一级页面，例如：

- `chat_page.py`
- `task_page.py`
- `supervision_page.py`
- `mistake_review_page.py`
- `score_trend_page.py`
- `nightly_review_page.py`
- `problem_board_page.py`
- `fortune_page.py`
- `settings_page.py`

组件文件负责页面内部模块，例如任务面板、错题面板、问题板面板。

UI 层规则：

- 可以调用 workflow。
- 不直接写 SQL。
- 不直接调用 LLM。
- 不拼复杂 prompt。
- 不把未校验的 LLM 输出写入数据库。

## Workflow 层

位置：

```text
src/kaoyan_agent/workflows/
```

Workflow 是应用编排层，负责把 UI、Agent、Repository、Schema 串起来。

主要文件：

- `chat_workflow.py`：在线聊天链路。
- `nightly_memory_workflow.py`：晚间回顾、问题发现、记忆更新链路。
- `workspace_workflow.py`：任务、错题、成绩、问题板等通用读取和状态更新。
- `settings_workflow.py`：设置页需要的记忆、模型和数据库信息。
- `planning.py`：学习任务、错题卡、激励任务等生成。
- `focus.py`：督学/专注记录。

Workflow 层可以协调多个 repository，也可以调用 agent，但不要把 SQL 分散到 UI 或 Agent 中。

## Agent 层

位置：

```text
src/kaoyan_agent/agents/
```

Agent 封装推理能力：

- `chat_agent.py`：对话回复。
- `nightly_memory_agent.py`：晚间结构化复盘。
- `problem_discovery_agent.py`：问题发现的最小 agent 接口。
- `practice_review.py`：错题卡生成。
- `motivation.py`：轻量激励和小任务。
- `query_rewriter.py`：检索查询改写。
- `router.py`：意图路由。

Agent 层规则：

- 可以调用 LLM client。
- 可以读取 prompt registry。
- 不直接写数据库。
- 输出影响关键业务表时，必须经过 schema 校验。

## Repository 层

位置：

```text
src/kaoyan_agent/repositories/
```

Repository 是 SQLite CRUD 边界。所有 SQL 应尽量放在这里。

关键 repository：

- `conversation_repository.py`：会话和消息。
- `memory_repository.py`：长期记忆。
- `problem_repository.py`：问题板。
- `nightly_review_repository.py`：晚间回顾记录。
- `study_tasks.py` / `study_task_repository.py`：学习任务。
- `practice_review.py` / `mistake_review_repository.py`：错题复习。
- `score.py` / `score_repository.py`：成绩记录。
- `focus.py` / `supervision_repository.py`：督学和专注记录。

`project_repository.py` 仍存在，但只用于历史数据库兼容。不要把它接回 UI 导航，也不要让用户看到多空间管理。

Repository 层规则：

- 只做 CRUD。
- 不调用 LLM。
- 不拼 prompt。
- 写入 `problem_board` 和 `memories` 的 LLM 数据必须来自 Pydantic 校验后的 `model_dump()`。

## DB 层

位置：

```text
src/kaoyan_agent/db/
```

主要文件：

- `schema.sql`：SQLite 表结构。
- `database.py`：连接、初始化、兼容迁移、通用 helper。
- `connection.py`：连接相关兼容文件。
- `migrations.py`：迁移入口。

真实数据库默认是：

```text
data/app.db
```

注意：

- 不删除 `data/app.db`。
- 不清空真实数据。
- 不提交真实 `.env`。
- 需要测试写库时使用临时 SQLite 数据库。

数据库里可能仍保留 `projects/project_id`，这是历史兼容字段。当前产品不把它作为用户可见架构。

## Pydantic Schema 层

位置：

```text
src/kaoyan_agent/schemas/
```

核心文件：

- `nightly_memory.py`：晚间结构化输出。
- `contracts.py`：通用数据契约。

晚间链路必须保持：

```text
LLM raw response
-> NightlyMemoryUpdateOutput.model_validate_json()
-> typed object
-> model_dump()
-> repositories
-> SQLite
```

解析失败时：

- 保存 `raw_response`。
- 保存 `parse_status="failed"`。
- 保存 `error_message`。
- 不写入 `problem_board`。
- 不写入 `memories`。
- UI 显示失败提示，不让 Streamlit 崩溃。

## Prompt 层

位置：

```text
src/kaoyan_agent/prompts/
```

当前结构：

```text
prompt_registry.py
nightly_memory_update_prompt.txt
```

`prompt_registry.py` 负责读取 prompt。真实 prompt 文本已经统一放进正式包内，不再使用根目录 `prompts/` 作为业务目录。

## LLM 服务层

位置：

```text
src/kaoyan_agent/services/llm_client.py
```

职责：

- 读取 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`。
- 提供 OpenAI-compatible 调用。
- 提供 LangChain-ready 的 `create_langchain_model()` 接口，`create_chat_model()` 作为兼容别名保留。
- 如果 LangChain 相关包不存在，继续使用现有 fallback，不强制引入额外依赖。

## Tests 为什么在根目录

`tests/` 保留在根目录是正常的 Python 项目结构：

```text
src/kaoyan_agent/   正式业务包
tests/              测试代码
```

这样测试不会被当成正式包安装进去，也更容易用：

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests
```

## 新功能开发流程

新增一个用户功能时，按这个顺序写：

1. 如果需要页面，先在 `ui/` 新增 page 或 component。
2. 在 `workflows/` 写业务编排。
3. 如果要调 LLM，在 `agents/` 写 Agent。
4. 如果要持久化，在 `repositories/` 写 CRUD。
5. 如果 LLM 输出要进关键表，在 `schemas/` 加 Pydantic schema。
6. 如果要改数据库，在 `db/schema.sql` 和迁移 helper 中兼容新增字段。
7. 在 `tests/` 增加测试。

## 不要这样写

- 不要在 UI 中直接 `sqlite3.connect()`。
- 不要在 UI 中直接调用 LLM。
- 不要在 Repository 中写 prompt 或调用 LLM。
- 不要把 LLM 原始 JSON 直接写入问题板或长期记忆。
- 不要恢复根目录 `agents/`、`db/`、`services/`、`ui/` 等旧业务包。
- 不要把 `projects/project_id` 做成新的用户可见产品入口。
- 不要删除或重建 `data/app.db`。

## 常用验证命令

```powershell

.\.venv\Scripts\python.exe -m unittest discover -s tests
.\.venv\Scripts\python.exe -c "from kaoyan_agent.db import init_db; init_db(); print('ok')"
.\.venv\Scripts\python.exe -c "from kaoyan_agent.prompts.prompt_registry import get_nightly_memory_prompt; print(get_nightly_memory_prompt()[:20])"
```
